# vgz2midi 드래그 앤 드롭 변환기

`vgz2midi.py`는 `.vgz`와 `.vgm` 파일을 편집·채보·편곡용 표준 MIDI 파일로 변환합니다.

## 가장 간단한 사용법

1. Windows에 Python 3를 설치합니다.
2. 변환할 `.vgz` 또는 `.vgm` 파일을 하나 이상 선택합니다.
3. 선택한 파일을 `vgz2midi.py` 위에 끌어다 놓습니다.
4. 원본 파일과 같은 폴더에 같은 이름의 `.mid` 파일이 생성됩니다.

예:

- 입력: `music.vgz`
- 출력: `music.mid`

폴더를 `vgz2midi.py` 위에 끌어다 놓으면 하위 폴더까지 검색하여 모든 `.vgz`와 `.vgm` 파일을 변환합니다.

## BPM 자동 감지

VGM 파일에는 일반적으로 BPM 값이 직접 저장되어 있지 않습니다. `vgz2midi.py`는 다음 정보를 조합하여 BPM을 추정합니다.

- 추출된 음표와 샘플 트리거의 시작 시점
- 이벤트 시작 간격의 반복 패턴
- 여러 채널이 동시에 발음되는 강세
- VGM 루프 길이 정보

감지한 BPM은 MIDI의 Tempo 메타 이벤트에 기록됩니다. VGM 샘플 위치를 감지한 템포 기준의 MIDI 틱으로 환산하면서 원곡의 전체 재생 시간을 유지합니다.

콘솔에는 감지된 BPM과 신뢰도가 표시됩니다. 이벤트가 적은 곡, 자유 박자 곡, 앰비언트 곡 또는 샘플 중심 곡은 반 BPM이나 두 배 BPM으로 판정될 수 있습니다.

BPM을 직접 지정하려면 다음과 같이 실행합니다.

```text
python vgz2midi.py music.vgz --bpm 128
```

`--bpm`을 생략하면 자동 감지를 사용합니다. 분석 가능한 이벤트가 너무 적으면 120 BPM을 기본값으로 사용합니다.

## 명령행 사용법

파일 하나 변환:

```text
python vgz2midi.py music.vgz
```

여러 파일 변환:

```text
python vgz2midi.py song1.vgz song2.vgm
```

폴더와 하위 폴더의 모든 파일 변환:

```text
python vgz2midi.py "D:\vgm music"
```

출력 폴더 지정:

```text
python vgz2midi.py music.vgz -o "D:\midi"
```

## PCM 및 ADPCM 변환 방식

표준 MIDI에는 원본 PCM 또는 ADPCM 파형을 넣을 수 없습니다. 따라서 변환기는 VGM 명령에서 확인할 수 있는 시퀀싱 정보를 최대한 보존한 상징적인 MIDI 샘플 트리거 이벤트를 생성합니다.

다음 정보를 보존하거나 추정합니다.

- 샘플 재생 시작 시점
- 샘플 정지 시점 또는 추정 재생 길이
- 데이터 뱅크 블록, ROM 주소 범위, 웨이브 번호 또는 DPCM 주소를 이용한 샘플 구분
- 재생률·주파수 레지스터가 존재할 때 상대적인 MIDI 음높이
- 레벨·감쇠값을 이용한 MIDI 벨로시티
- 고정 리듬 보이스의 General MIDI 드럼 매핑

### 구현된 샘플 재생 경로

- **YM2612 DAC PCM**
  - DAC 레지스터 `0x2A` 직접 쓰기
  - VGM PCM 뱅크 명령 `0x80`~`0x8F`
  - PCM 위치 이동 명령 `0xE0`
- **VGM DAC 스트림 시스템**
  - 데이터 블록과 데이터 뱅크
  - 스트림 설정·재생·정지 명령 `0x90`~`0x95`
  - 비압축 뱅크와 지원 가능한 n-bit·DPCM 압축 뱅크
- **YM2608 / OPNA**
  - 고정 리듬 ROM 6보이스
  - Delta-T ADPCM-B의 주소 범위, 레벨, 재생률, 반복 및 정지 추적
- **YM2610 / OPNB**
  - ADPCM-A 6보이스
  - Delta-T ADPCM-B 재생
- **Y8950**
  - Delta-T ADPCM 재생
- **NES APU**
  - DPCM 주소, 길이, 재생률, 반복 상태 및 직접 출력 레벨
- **HuC6280**
  - DDA 샘플 쓰기 구간과 쓰기 속도 추정
- **YMF278B / OPL4**
  - PCM 24보이스의 키온·키오프
  - 웨이브 번호, F-number, 옥타브 및 Total Level 추적

멜로디형 샘플 트랙에는 중립적인 샘플 계열 MIDI 프로그램을 사용합니다. 대부분의 VGM 샘플 형식에는 음악적인 루트 키가 없으므로, 동일 샘플에서 처음 관측된 재생률을 C4로 두고 이후 재생률 변화를 상대 음정으로 매핑합니다. 드럼형 샘플과 고정 리듬 샘플은 MIDI 퍼커션 채널에 배치합니다.

## 칩별 일반 음원 지원 범위

### Game Boy DMG APU

- 사각파 2채널
- 웨이브 1채널
- 노이즈 1채널을 MIDI 드럼으로 변환
- 트리거 시점과 레지스터 기반 볼륨 추출

웨이브 RAM 변경은 음색 정보이며 독립적인 음표 이벤트가 아닙니다. 하드웨어 길이 카운터 종료, 스윕 및 엔벌로프 변화는 정확히 시뮬레이션하지 않습니다.

### NES APU

- 펄스파 2채널
- 삼각파 1채널
- 노이즈 1채널을 MIDI 드럼으로 변환
- DPCM 샘플 트리거

확장 음원은 현재 해석하지 않습니다.

### HuC6280 PSG

- 웨이브테이블 톤 6채널
- 5·6번 채널의 노이즈를 MIDI 드럼으로 변환
- DDA PCM 구간
- 채널 선택, 주파수, 볼륨 및 활성화 상태 추적

LFO 동작은 재현하지 않습니다.

### Konami K051649 SCC

- 웨이브테이블 5채널
- 주파수, 볼륨 및 키 마스크 추적

웨이브폼 RAM은 음색에 영향을 주지만 MIDI에 삽입하지 않습니다.

### Atari POKEY

- 4채널
- 일반 클럭, 고속 클럭 및 채널 결합 모드의 음정 추정
- 폴리노미얼·노이즈 모드를 MIDI 드럼으로 변환

디스토션 모드와 카운터 경계 동작은 사이클 단위 에뮬레이션이 아닌 근사 변환입니다.

### Philips SAA1099

- 톤 6채널
- 두 노이즈 발생기의 채널 라우팅을 MIDI 드럼으로 변환
- 진폭, 옥타브, 주파수 및 활성화 레지스터 추적

하드웨어 엔벌로프는 재현하지 않습니다.

## 변환 한계

VGM/VGZ는 MIDI 음표가 아니라 사운드 칩 명령을 저장합니다. 생성된 MIDI는 오디오 변환 결과가 아니라 분석과 편집을 위한 근사 결과입니다.

- 원본 PCM·ADPCM·웨이브폼 ROM·웨이브폼 RAM의 소리는 MIDI 파일에 포함되지 않습니다.
- VGM 데이터에는 샘플의 원래 악기 이름과 음악적인 루트 키가 없는 경우가 많으므로 샘플 트랙 이름과 음정 매핑을 추정합니다.
- 바이트 길이와 재생률로 종료 시점을 계산할 수 있지만, 칩의 반복·상태 플래그 또는 외부 ROM 동작에 따라 실제 길이와 다를 수 있습니다.
- FM 음색, 세부 엔벌로프, 변조, 피치 슬라이드, 하드웨어 효과 및 특수 채널 모드는 표준 MIDI로 정확히 표현되지 않을 수 있습니다.
- FM 채널 3 특수 모드의 슬롯별 독립 음정과 CSM 동작은 완전히 해석하지 않습니다.

## 오류 로그

변환 중 오류가 발생하면 `vgz2midi.py` 옆에 `vgz2midi_errors.txt`가 생성됩니다.
